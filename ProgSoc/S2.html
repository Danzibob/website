Variables

A variable is the way a program stores data. They are common across almost all langauges. In JavaScript you create variables as follows: 

var bannana = 4
var appple = "red"

The var part is only needed when you first declare the variable. When you use the variable it's not neccessary:

console.log(apple)
console.log(bannana * 2)
//console output:
    > "red"
    > 8

Another concept shared by many high level languages is that of global and local variables. In JavaScript, if you declare a variable inside a function, it's local, but if you declare it outside of the function, it's global. Global variables can be accessed anywhere in the code, but local can only be accessed from within the function they were created.

Lists

Lists in JavaScript (aka Arrays) are similar to python but different to Java, in that they can contain any number of any sort of item. For example:

var my_list = [1,2,3,"a","b",["inception","bwaaa"]]

To get at the data in the list we do the following:

console.log(my_list[0])
    > 1
console.log(my_list[4])
    > "b"
console.log(my_list[5][1])
    > "bwaaa"

To put data into the list we use list.push(x):

var numbers = [2,4,6]
numbers.push(8)
console.log(numbers)
    > [2,4,6,8]

And to remove it we use list.splice(i,n). i is the index to start at and n is the number of items to remove:

var numbers = [2,4,6,8,10,12]
numbers.splice(3,1)
console.log(numbers)
    > [2,4,6,10,12]
numbers.splice(1,3)
console.log(numbers)
    > [2,12]

Loops

One of the most common uses of loops is to perform an operation on every element in a list. The syntax of a for loop in JavaScript is as follows:

for(var i = 0; i < 10; i++){
    //do stuff
}

This may look a little daunting at first, but this syntax can be broken down into simpler pieces. Firstly var i = 0 creates a new variable called i. The next line is the condition to see if the loop should keep running.

if(i < 10){
    //keep running
} else {
    //end the loop
}
//spoiler this is how if statements work

Finally the i++ simply adds 1 to i at the end of each iteration.

The semicolons seperate each section so they are treated as different lines of code
