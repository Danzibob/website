The Canvas

The canvas is the main part of any p5 sketch. It's where everything is drawn. The code you see to the left consists of two parts: setup and draw.

  • setup only runs once - as soon as the page is loaded
  • draw runs 60 times a second, or as fast as it can go below that

The createCanvas function does what it says on the tin - creates a canvas. The two numbers are the X and Y dimensions of the canvas respectively, so this canvas would be 400px wide and 300px tall. By default the canvas is blank. The next step is to add color. Click the tab above to move on.

Color

The way computers display color is through lighting up Red, Green and Blue LEDs in each pixel to produce the desired color. Light mixes differently to the color system you may be used to. Try it out below:

Because of this system, the way we tell a computer a color is with a list of 3 numbers, corresponding to the Red, Green and Blue, hence RGB. These numbers are usually between 0 and 255, but are also often in hexadecimal (eg #6D2B6A). But in p5 we can stick to normal numbers. For example, the purple we have above would be (109,43,106)

In p5 many functions use color as an argument, for example:

  • background(r,g,b) - covers the whole canvas with the specified color
  • fill(r,g,b) - specifies the color that new shapes will be filled with
  • stroke(r,g,b) - specifies the edge color of new shapes

Shapes

p5 has several basic functions for drawing shapes, the main ones being ellipse, rect, and line

  • line takes the arguments (x1, y1, x2, y2), drawing a line from point 1 to point 2. Here the p5 keywords width and height are used to make the line extend from the top right (width, 0) to the bottom left (0, height)
  • rect takes the arguments (x, y, w, h), placing the top left corner of the shape at (x,y) and creating a rectangle that's w wide and h tall
  • ellipse also takes the arguments (x, y, w, h). This puts the centre of the ellipse at (x,y) then stretches it out to the specified width and height.

Remeber: Coordinates in p5 start at the top left, so Y increases as you go down

Styling

How a shape looks is changed by the sytling options you choose. The main options are:

  • fill(r,g,b) Changes the fill color of the shape
  • noFill() Turns off fill for the shape
  • stroke(r,g,b) Changes the color for the shape border
  • strokeWeight(w) Changes the width of the border
  • noStroke() Turns off the shape border

The Mouse

p5 has two keywords for reading the position of the mouse, namely mouseX and mouseY. Theres also a function called mousePressed that triggers on every click.

Here, both keywords are being used to tell the program to draw an ellipse around the mouse, which is a standard use of the mouse position. However, mouseX is also being used here to change the color of the ellipse; when mouseX increases, g increases too.

However, the circles aren't being cleared away, because the background function that usually does this job is only triggered when the mouse is pressed.



NB: the map function simply scales the mouseX to stay between 0 and 255
